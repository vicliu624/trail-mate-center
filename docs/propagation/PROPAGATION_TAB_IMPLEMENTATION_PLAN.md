# TrailMateCenter 山地 LoRa 专业级规划与部署决策系统方案（升级版）

## 1. 系统定义与目标

我们定义的不是“传播可视化工具”，而是一个可用于山地通信规划、部署评估和运行优化的专业系统：

> 山地 LoRa 专业级规划与部署决策系统

系统目标：

- 预测级：输出山地场景下可解释的覆盖预测结果。
- 决策级：输出可执行的中继部署建议与多方案对比。
- 网络级：输出干扰、容量、可靠性等网络运行指标。
- 学习级：结合实测数据持续校准模型参数，形成可演进能力。

## 2. 架构原则

- 物理模型与渲染彻底解耦。
- 传播模型可插拔（支持模型升级与并行验证）。
- 统计模型可校准（支持实测反演参数）。
- 计算与展示分层（可替换渲染引擎，不影响核心算法）。
- 结果可追溯（输入、模型版本、输出全量可审计）。
- 2D 分析工作台为唯一目标交互架构；不再以 Unity 作为传播产品依赖。

## 4. 环境建模层升级

从单一 DEM 升级为三层环境模型。

## 4.1 地形模型（DEM）

- 数据源：SRTM 30m、ALOS 12.5m、本地 GeoTIFF。
- 能力：LOD 金字塔、坡度与曲率计算、山脊识别输入。
- 结果：提供高程采样、剖面采样、地形派生特征（坡度、曲率、遮挡角）。

## 4.2 植被衰减模型（新增）

引入土地覆盖数据（森林、灌木、草地、岩石、城市等），沿路径累计植被穿越距离。

经验模型：

```text
L_veg = α × d_veg
```

其中：

- `α`：单位长度衰减系数（dB/m）。
- `d_veg`：路径穿越植被长度（m）。

典型系数建议（初始值，后续可校准）：

- 稀疏林：`0.2 ~ 0.4 dB/m`
- 密林：`0.5 ~ 1.5 dB/m`

实现方式：

- 地表栅格叠加（land cover raster）。
- 射线路径逐段采样并累计各类地物长度。

## 4.3 地表反射模型（科研级接口）

用于多径场景的简化反射建模。

- 输入：地表类别对应介电常数或经验反射系数 `Γ`。
- 初版：按地表类型使用经验常数。
- 进阶：支持入射角频段相关反射系数查表。

## 5. 传播物理模型引擎（升级版）

统一链路预算形式：

```text
Pr = Pt + Gt + Gr
     - FSPL
     - DiffractionLoss
     - VegetationLoss
     - ReflectionLoss
     - ShadowFading
     - OtherLoss
```

说明：`OtherLoss` 可包含线损、连接器损耗、设备实现损耗。

## 5.1 基础传播项

- 距离项：地表距离 + 高程影响路径分段。
- 自由空间损耗：
  `FSPL(dB) = 32.44 + 20log10(f_MHz) + 20log10(d_km)`
- 视距判定：DEM 剖面采样 + 直线遮挡检测。
- 绕射损耗：单刀锋模型（ITU 近似）。
- 菲涅尔区校正：按侵入比例施加附加损耗。

## 5.2 双向链路分析（新增）

必须分别计算：

- 上行（终端 -> 网关/中继）
- 下行（网关/中继 -> 终端）

原因：

- 发射功率可能不同。
- 天线高度可能不同。
- SF 与带宽策略可能不同。

输出：

- `Uplink Margin`
- `Downlink Margin`
- `Link Feasible`（双向都满足时为 true）

## 5.3 衰落裕量（Fade Margin）

定义：

```text
Margin = Pr - Sensitivity
```

建议分级：

- `< 5 dB`：危险
- `5 ~ 10 dB`：边缘
- `> 15 dB`：稳定

界面要求：

- 颜色编码（红/黄/绿）。
- 数值标签（dB）。

## 5.4 覆盖概率模型（新增）

覆盖判定不再是二值可达，而是概率可达。

假设阴影衰落为对数正态：

```text
Pr ~ N(μ, σ)
```

山地场景可用初值：`σ = 6 ~ 10 dB`。

计算：

```text
P(Pr > Sensitivity)
```

输出图层：

- 95% 可达区
- 80% 可达区
- 平均覆盖图
- 可靠性覆盖图

## 6. 网络级仿真与优化模块（专业级）

## 6.1 多节点干扰模拟

对每个网格点计算：

```text
SINR = S / (I + N)
```

其中：

- `S` 为目标信号。
- `I` 为其他节点同频干扰叠加。
- `N` 为噪声底。

LoRa 相关策略：

- 同频同 SF：按完全冲突处理。
- 同频不同 SF：按 SF 捕获矩阵处理（可配置阈值）。

输出：

- 干扰热力图
- 冲突概率图
- SINR 分布统计

## 6.2 网络容量估算

估算输入：

- 占空比限制
- 节点数
- 平均报文周期
- 报文时长（受 SF/BW/CR 影响）

ALOHA 近似：

```text
Throughput ≈ G × e^{-2G}
```

输出：

- 最大支持节点数
- 安全运行区间
- 过载阈值预警

## 6.3 中继部署优化（升级版）

目标函数：

```text
Score =
  w1 × 覆盖面积
+ w2 × 覆盖概率
- w3 × 盲区面积
- w4 × 干扰水平
```

求解算法分级：

- 基础：贪心算法
- 进阶：遗传算法
- 科研：模拟退火

输出：

- TopN 候选方案
- 每个方案的评分分解
- 关键权衡（覆盖提升 vs 干扰增加）

## 6.4 山脊自动识别

基于 DEM 提取中继候选点：

- 局部极值检测
- 曲率分析
- 坡度阈值过滤

结果：

- 候选点清单
- 3D 高亮显示
- 与禁入区/不可部署区联动过滤

## 7. 科研级增强模块

## 7.1 多径反射建模

每条路径计算一次地面反射分量：

- 直达路径与反射路径长度差
- 相位差近似
- 反射系数 `Γ`

场强叠加近似：

```text
E_total = E_direct + Γ × E_reflect
```

用途：

- 识别深衰落区与增强区
- 作为调试和研究模式（默认可关闭）

## 7.2 蒙特卡洛传播仿真

随机扰动参数：

- 植被密度
- 地表参数
- 阴影衰落

执行 `N` 次仿真，输出：

- 置信区间
- 覆盖稳定性指数
- 规划方案鲁棒性排序

## 10. 传播页完整交互设计（生产级）

推荐布局：`左（参数）-中（2D/3D）-右（分析）-下（任务）`。

## 10.1 左侧参数区

- 无线参数：频率、带宽、SF、CR、功率、增益、线损。
- 节点参数：发射/接收天线高度、上行下行策略。
- 场景参数：AOI、分辨率、DEM/地表/植被数据源。
- 模型参数：绕射、菲涅尔、阴影衰落、反射、蒙特卡洛开关。
- 优化参数：目标函数权重、候选点约束、求解算法。

## 10.2 中央可视区

- 2D 图：地图叠加覆盖、概率、干扰、容量等图层。
- 3D 图：Unity 场景联动展示。
- 交互：点选任意位置展示单点预算与双向裕量。

## 10.3 右侧分析区

- 单点预算卡
- 双向链路卡
- 概率可靠性卡
- 干扰与容量卡
- 模型假设与风险提示卡

## 10.4 下方任务区

- 任务队列、进度、耗时
- 缓存命中、模型版本、数据版本
- 失败重试、暂停恢复
- 导出报告（GeoTIFF/PNG/CSV/JSON）

## 11. 插件化模型接口（可扩展关键）

```csharp
public interface IPropagationModel
```

收益：

## 12. 性能架构设计

必须采用：

- 并行计算（CPU 多核）。
- GPU 加速（Compute Shader，用于大规模栅格后处理）。
- 分块计算（Tile-based）。
- 分层缓存（静态、半静态、动态）。

分层缓存定义：

- 静态层：地形、地表、植被底图与派生特征。
- 半静态层：频段固定下的部分传播中间量。
- 动态层：节点变化后的实时仿真结果。

建议性能目标：

- `10km x 10km @ 50m` 在标准 8 核 CPU 上完成基础覆盖计算 `< 3s`。
- 网络级仿真在同尺度下控制在 `< 10s`（不含蒙特卡洛）。

## 15. 验收标准

- 可解释性：单点结果可完整追踪损耗分解。
- 决策性：可输出可执行的 TopN 中继方案。
- 网络性：可输出干扰与容量风险图。
- 稳定性：任务可取消、可恢复、可复算。
- 可校准性：导入实测后误差可量化下降。

## 17. 实施建议（避免完美主义陷阱）

建议执行顺序：

1. 先完成“物理层 + 双向链路 + 植被 + 裕量”。
2. 再完成“网络级干扰 + 容量 + 中继优化”。
3. 最后做“多径反射 + 蒙特卡洛 + 数据校准”。

这条路径能保证每个阶段都有可交付成果，同时不断抬高系统上限。
