# 07 双向链路裕量

## 1. 核心理论

LoRa 链路可用性必须考虑双向：

- 上行：终端 -> 网关/中继
- 下行：网关/中继 -> 终端

定义：

```text
Margin = Pr - Sensitivity
```

链路可行条件：上行和下行均满足最小裕量阈值。

## 2. 在系统中的用途

### 2.1 作用点

- 解决“下行可达、上行不可达”这类单向假覆盖问题。
- 右侧 Link Card 同时显示 Downlink/Uplink。

### 2.2 输入输出

输入：上下行参数（功率、天线、SF/BW）。  
输出：`UplinkMargin`、`DownlinkMargin`、`LinkFeasible`。

## 3. 用在哪一方

- 数据层：不直接使用
- 传播计算层：双向计算逻辑
- 网络仿真层：上行/下行业务成功率评估
- 优化决策层：用于中继方案约束
- UI/可视化层：双向指标与颜色告警
- 校准学习层：可分上下行独立校准

## 4. 实现注意

- 切勿复用同一参数做上下行。
- 需支持不同 SF 策略与不同灵敏度表。

## 5. 验收要点

- 单向失败时 `LinkFeasible=false`。
- UI 必须同时显示上下行读数。
