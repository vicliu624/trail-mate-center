# 10 ALOHA 网络容量估算

## 1. 核心理论

LoRa 网络常用 ALOHA 近似估算容量上限：

```text
Throughput ≈ G * e^(-2G)
```

`G` 是 offered load（单位时间内尝试发送强度）。

## 2. 在系统中的用途

### 2.1 作用点

- 给出“安全运行区间”。
- 输出最大支持节点数估算。

### 2.2 输入输出

输入：节点数、发包周期、报文长度、占空比限制。  
输出：吞吐估算、容量阈值、过载告警。

## 3. 用在哪一方

- 数据层：不直接使用
- 传播计算层：不直接使用
- 网络仿真层：核心
- 优化决策层：容量约束输入
- UI/可视化层：Network Metrics 卡片
- 校准学习层：可与实网吞吐对比修正

## 4. 实现注意

- 该公式是近似上界，不是精确协议模拟器。
- 应结合 SF 分布与实际报文时长修正 G。

## 5. 验收要点

- 负载上升到某阈值后吞吐下降（出现拐点）。
- 容量告警阈值随参数变化而变化。
