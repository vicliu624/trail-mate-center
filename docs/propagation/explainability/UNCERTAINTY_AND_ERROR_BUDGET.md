# UNCERTAINTY_AND_ERROR_BUDGET

> 目的：把不确定性来源拆清楚，避免把“模型偏差”误当“部署问题”。

## 1. 不确定性来源分类

## 1.1 数据不确定性

- DEM 分辨率限制
- 土地覆盖分类误差
- 地表参数时效性不足

## 1.2 模型不确定性

- 单刀锋简化误差
- 线性植被损耗近似误差
- 固定 sigma 误差

## 1.3 参数不确定性

- alpha_class 未校准
- sigma 未分区
- 反射系数经验设定

## 1.4 观测不确定性

- GPS 误差
- RSSI 测量波动
- 时间不同步

## 2. 误差预算表示法

建议将总误差表达为：

```text
E_total^2 ≈ E_data^2 + E_model^2 + E_param^2 + E_measure^2
```

用于报告解释时，给出占比估计（区间即可）。

## 3. 输出指标

- `prediction_mae_db`
- `prediction_rmse_db`
- `uncertainty_p50/p90`
- `dominant_error_source`

## 4. UI 呈现要求

- 在 `Coverage Probability` 卡片旁显示置信提示。
- 对低置信区域叠加纹理层（而非实色）。
- 导出报告必须有“误差预算”章节。

## 5. 迭代策略

1. 先收敛数据误差（对齐与版本控制）。
2. 再收敛参数误差（校准 alpha/sigma）。
3. 再补模型误差（多障碍绕射、多径增强）。

## 6. 用在哪一方

- 数据层：识别数据质量风险
- 传播计算层：输出不确定性字段
- 网络仿真层：容量/干扰结果附带置信度
- 优化决策层：评分加入鲁棒性项
- UI/可视化层：展示置信纹理与告警
- 校准学习层：定义优先校准顺序
