# CALIBRATION_AND_VALIDATION_PROTOCOL

> 目的：规范实测校准流程，确保参数更新有证据、可复验。

## 1. 数据采集协议

## 1.1 必采字段

- 时间戳（UTC）
- 发射点/接收点坐标与高度
- 频率、SF、BW、CR、发射功率
- RSSI、SNR、链路方向（上/下行）
- 环境标签（林地/灌木/开阔/城镇）

## 1.2 采样建议

- 同场景至少 20 条链路样本。
- 覆盖近/中/远距离段。
- 保留重复测量用于波动估计。

## 2. 数据切分

- 训练集：70%
- 验证集：15%
- 测试集：15%

要求：按地形/地类分层抽样，避免单场景过拟合。

## 3. 校准流程

1. 载入当前参数集（seed）。
2. 在训练集拟合 alpha_class 与 sigma。
3. 在验证集调参与早停。
4. 在测试集报告最终 MAE/RMSE。
5. 生成 `calibration_run_id` 与参数版本。

## 4. 验收阈值（建议）

- MAE 下降 >= 10%
- RMSE 下降 >= 8%
- 测试集误差不劣化

## 5. 防过拟合规则

- 不允许仅报告训练集改善。
- 参数变更必须附带测试集报告。
- 回归后对历史基准场景做回放验证。

## 6. 输出工件

- `calibration_report.json`
- `paramset_new.json`
- `before_vs_after_metrics.csv`

## 7. 用在哪一方

- 数据层：管理实测数据与标签
- 传播计算层：加载新参数重新推理
- 网络仿真层：继承校准后的链路输入
- 优化决策层：提高推荐可信度
- UI/可视化层：展示校准前后对比
- 校准学习层：执行核心流程
