## 3. 输出格式建议

- UI：右侧 `Analysis & Results` 卡片简版。
- 报告：导出完整模板（Markdown/PDF）。
- API：结构化 JSON 字段。

## 4. 用在哪一方

- 优化决策层：输出解释链
- UI/可视化层：展示推荐依据
- 校准学习层：对比推荐稳定性
- 项目管理/验收：作为评审材料

## 5. 验收要求

- 每个 TopN 方案必须有评分分解。
- 每个推荐必须附带至少 1 条风险提示。
- 报告可独立阅读，不依赖源码上下文。
