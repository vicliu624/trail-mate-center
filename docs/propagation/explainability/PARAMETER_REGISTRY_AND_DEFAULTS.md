# PARAMETER_REGISTRY_AND_DEFAULTS

> 目的：统一参数语义、默认值、范围、来源与责任归属，避免口径漂移。

## 1. 参数治理原则

- 每个参数必须有：单位、默认值、范围、来源依据、敏感度级别。
- 参数变更必须版本化：`parameter_set_version`。
- 参数分层：无线参数 / 环境参数 / 统计参数 / 网络参数 / 优化参数。

## 3. 参数版本与发布规则

- 参数集命名：`paramset_YYYYMMDD_xx`。
- 每次发布记录：修改原因、影响范围、回归结果。
- 高敏感参数（alpha/sigma/weights）必须附带校准证据。

## 4. UI 落地要求

- 仅暴露“常用参数”在快速区。
- 高风险参数放入 `Advanced` 并显示说明。
- 参数越界时禁止 Run，并给出修正建议。

## 5. 验收要求

- 所有计算参数都能在 registry 中找到定义。
- 导出结果包含 `parameter_set_version` 和参数哈希。
